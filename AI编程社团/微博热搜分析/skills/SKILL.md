# Claude Slash Commands 技能

## 概述

这是一套通用型斜杠命令系统，提供 11 种核心工作流程，覆盖软件开发全流程。

**核心原则**：
- 简洁：单命令完成单一任务
- 智能：自动识别项目类型和上下文
- 安全：执行前确认，保留用户控制权

---

## 详细命令说明

### 代码分析 (/a)

**功能**：分析代码库结构、技术栈、依赖关系、识别潜在问题

**参数**：
- 无参数：快速分析技术栈和目录结构
- `full` 或 `完整`：完整分析，包括依赖关系、代码量统计
- `dep` 或 `依赖`：专注分析依赖关系
- `file` 或 `文件`：分析文件结构和模块划分

**工作流程**：
1. 扫描项目根目录，识别技术栈（package.json、pyproject.toml 等）
2. 分析目录结构，理解模块划分
3. 检查依赖项和版本
4. 生成分析报告

**使用示例**：
```
/a 快速分析
/a full 完整分析
/a 依赖 看看有哪些依赖
```

### 项目构建 (/b)

**功能**：运行构建命令、安装依赖、类型检查、代码格式化

**参数**：
- 无参数：自动识别并运行构建命令
- `install` 或 `安装`：安装依赖
- `type` 或 `类型`：运行类型检查
- `lint` 或 `格式`：运行代码格式化/检查
- `watch` 或 `监听`：监听模式构建

**工作流程**：
1. 检测项目类型（Node.js、Python、Rust 等）
2. 选择合适的构建命令
3. 执行用户指定的操作
4. 报告结果

**使用示例**：
```
/b
/b install
/b type
/b lint
```

### Git 提交 (/c)

**功能**：分析变更、生成规范的 commit message、执行提交

**参数**：
- 无参数：分析变更并生成 commit message
- `feat` / `fix` / `docs` / `style` / `refactor` / `test` / `chore`：指定类型前缀
- `push` 或 `推送`：提交后自动推送到远程

**工作流程**：
1. 运行 `git status` 查看变更文件
2. 运行 `git diff` 详细分析每个文件
3. 根据变更内容生成规范的 commit message
4. 询问用户确认（可修改）
5. 执行 `git commit`
6. 如果指定 `push`，执行 `git push`

**提交规范**：
```
<type>(<scope>): <subject>

<body>

<footer>
```

**使用示例**：
```
/c
/c fix 修复登录bug
/c push 提交并推送
```

**参数**：
- 无参数：进入交互式生成模式
- `component` 或 `组件`：生成 UI 组件

**工作流程**：
1. 理解用户需求

```

### 代码审查 (/r)

**功能**：检查代码质量、安全漏洞、性能问题、风格一致性

**参数**：
- 无参数：审查所有变更文件
- `security` 或 `安全`：专注安全审查
- `perf` 或 `性能`：专注性能分析
- `style` 或 `风格`：检查代码风格
- `todo` 或 `待办`：检查 TODO/FIXME

**审查维度**：
- 代码质量：命名、复杂度、重复
- 安全漏洞：注入、XSS、认证、加密
- 性能问题：内存、算法、数据库查询
- 风格一致性：ESLint、Prettier、团队规范

**工作流程**：
1. 获取需要审查的代码（变更文件或指定文件）
2. 执行多维度分析
3. 生成审查报告（分为：错误、警告、建议）
4. 提供改进建议和修复代码

**输出格式**：
```
## 代码审查报告

### 严重问题 (需立即修复)
- [文件:行号] 问题描述
  建议修复方案

### 警告 (建议修复)
- [文件:行号] 问题描述
  建议修复方案

### 优化建议
- [文件:行号] 建议内容
```

**使用示例**：
```
/r 审查我的代码
/r security 检查安全漏洞
/r perf 分析性能问题
```

### 代码搜索 (/s)

**功能**：搜索文件、内容、API 定义、函数调用链

**参数**：
- 无参数：进入交互式搜索模式
- `file` 或 `文件`：按文件名搜索
- `content` 或 `内容`：按文件内容搜索
- `api` 或 `接口`：搜索 API 定义和使用
- `call` 或 `调用`：搜索函数调用链

**工作流程**：
1. 解析搜索关键词
2. 选择合适的搜索工具（grep、find、glob）
3. 执行搜索
4. 整理结果，按相关性排序
5. 显示匹配位置和上下文

**使用示例**：
```
/s functionName 搜索函数
/s "TODO" 文件
/s fetch 查找所有 API 调用
/s getUserData 调用链
```

**工作流程**：
1. 检测测试框架（Jest、Vitest、Pytest 等）
2. 执行用户指定的测试
3. 收集测试结果
4. 生成报告

**使用示例**：
```
/t 运行所有测试
/t unit 单元测试
/t coverage 生成覆盖率报告
```

### 微博热搜分析 (/w)

**功能**：抓取微博热搜、搜索背景信息、生成产品创意、输出 HTML 报告

**参数**：
- 无参数：抓取当前热搜，生成产品创意报告
- `trend` 或 `趋势`：分析热搜趋势
- `product` 或 `产品`：专注产品创意生成
- `report` 或 `报告`：生成详细分析报告

**工作流程**：
1. 抓取微博热搜榜
2. 搜索每个热搜的背景信息
3. 基于热搜生成产品创意
4. 输出 HTML 格式报告

**输出文件**：
- `weibo_analysis_report.html` - 热搜分析报告
- `weibo_product_ideas.html` - 产品创意报告

**使用示例**：
```
/w

/w report 生成详细报告
```

## 常见问题

**Q: 命令不识别怎么办？**
A: 确保在 Claude Code 环境中使用，检查 `.claude/commands.json` 是否存在。

**Q: 如何添加自定义命令？**
A: 编辑 `.claude/commands.json`，添加新的命令配置。

**Q: `/b` 命令如何识别项目类型？**
A: 根据项目根目录的特征文件识别：package.json (Node.js)、pyproject.toml (Python)、go.mod (Go) 等。

**Q: `/c` 生成的 commit message 不满意怎么办？**
A: 可以直接修改，Claude 会按你修改后的内容提交。

**Q: 可以在非项目目录使用吗？**
A: 部分命令（如 `/z`、`/x`）可以在任何目录使用，但 `/a`、`/r`、`/s` 等命令需要在项目目录中使用。

## 输出约定

- **成功**：绿色输出，显示结果摘要
- **警告**：黄色输出，提示潜在问题
- **错误**：红色输出，显示错误信息和解决建议
- **文件生成**：蓝色输出，显示生成的文件路径
